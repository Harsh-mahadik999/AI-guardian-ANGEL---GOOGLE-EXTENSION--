# 🔐 Security Policy

## 🛡️ Supported Versions

We actively maintain and provide security updates for the following versions of **Guardian AI**:

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | ✅ Supported       |
| 1.x     | ⚠️ Limited Support |
| < 1.0   | ❌ Not Supported   |

---

## 🚨 Reporting a Vulnerability

We take security seriously. If you discover a vulnerability, please report it responsibly.

### 📩 How to Report
- Email: mahadikhh@gmail.com 
- Or open a **private security advisory** on GitHub

### 📋 Please Include:
- Description of the issue
- Steps to reproduce
- Affected component(s)
- Screenshots or proof-of-concept (if applicable)
- Suggested fix (optional)

---

## ⏱️ Response Timeline

| Stage              | Timeframe        |
|--------------------|-----------------|
| Initial Response   | Within 24–48 hrs |
| Investigation      | 2–5 days         |
| Fix Deployment     | 3–10 days        |

We will keep you updated throughout the process.

## 🔒 Security Best Practices

Guardian AI follows these practices:

- Minimal permissions usage (Chrome Manifest V3)
- Secure API communication (HTTPS only)
- No storage of sensitive personal data
- Local processing wherever possible
- Regular dependency updates
- Input validation & sanitization

---

## ⚠️ Known Limitations

- AI-based analysis may not detect all threats
- Heuristic detection can produce false positives/negatives
- Browser DOM changes (e.g., Gmail UI updates) may affect detection accuracy
- Payment verification is advisory, not guaranteed protection

---

## 🔐 Data & Privacy

- Only necessary page content is analyzed
- AI requests are sent securely via encrypted APIs
- No user tracking or behavioral profiling
- No selling or sharing of user data
